# 课件可读化转换工具设计技术路线

## 1. 产品定义

面向课程课件和企业培训材料，将 PPT/PPTX 转换为适合阅读、批注和复习的 PDF。系统优先保留原生排版和图片，并根据动画顺序、遮挡风险和页面复杂度，生成动画导读、可选 AI 讲义解释和问题报告。

## 2. 用户和场景

| 用户 | 场景 | 主要价值 |
|---|---|---|
| 大学生 | 课后复习、导入笔记软件 | 看懂老师上课动态讲解顺序 |
| 企业员工 | 培训回看、入职学习 | 离开讲师也能理解流程和重点 |
| 老师/培训负责人 | 发放课件、沉淀材料 | 自动把演示稿变成可自学讲义 |

## 3. 输入范围

| 输入 | 首版支持 | 说明 |
|---|---|---|
| `.pptx` | 支持 | 首版唯一正式输入 |
| `.ppt` | 不支持 | 老格式后续再评估 |
| PDF | 不支持 | 已丢失 PPT 动画结构 |
| Keynote | 不支持 | 格式链路不同 |
| 加密 PPTX | 不支持 | 直接提示 |
| 含音视频 PPTX | 静态保留并提示 | 后续考虑转写、关键帧和摘要 |

## 4. 输出文件

| 文件 | 首版 | 用途 |
|---|---:|---|
| `base.pdf` | 必须 | 普通原生转换版，保留排版和图片 |
| `guide.pdf` | 必须 | 动画导读基础版 |
| `report.json` | 必须 | 动画、遮挡、不支持项、转换问题报告 |
| `preview.html` | 建议 | Web 端预览和调试 |
| `ai_notes.pdf` | 后续 | AI 讲义解释版 |

## 5. 输出模式

| 模式 | 输出 | 首版状态 |
|---|---|---|
| 标准版 | `base.pdf` | 做 |
| 动画导读版 | `guide.pdf` | 重点做 |
| AI 讲义版 | `ai_notes.pdf` | 后续做 |

首版不做单独批注版。核心目标是解决动画如何融入 PDF，而不是简单加留白页。

## 6. 动画融入策略

动画导读必须形象地融入 PDF 页面本身，不只是简单放在旁边说明。

| 形态 | 使用场景 | 做法 |
|---|---|---|
| 原生增强页 | 原页清晰、变化少 | 保留原页，叠加编号、箭头、高亮 |
| 融合重排页 | 动画影响理解、页面可优化 | 保留核心图片/图表，重排文字和变化顺序 |
| 少量拆分页 | 内容复杂、同页无法读清 | 按语义组拆成 2-4 页 |

融合重排是核心攻关点。目标是在原意不变的前提下，让改后排版协调、易懂、能体现动画顺序。

## 7. 视觉规则

| 规则 | 定法 |
|---|---|
| 字号 | 标注文字不小于正文的 80% |
| 颜色 | 一种强调色 + 灰色辅助 |
| 编号 | 动画顺序统一用 ①②③ |
| 箭头 | 只表示阅读顺序或流程 |
| 高亮 | 只圈变化对象 |
| 说明文字 | 每个变化点最多 1 句话 |

可用表达：嵌入式编号、动态轨迹线、局部高亮、半透明幽灵层、渐变淡化、局部放大框、阅读顺序线。

## 8. 二次排版触发条件

| 情况 | 是否二次排版 |
|---|---:|
| 页面无动画且排版清晰 | 不重排 |
| 只有 1-2 个小元素出现 | 不重排，只轻标注 |
| 多元素逐步出现但不拥挤 | 不重排，只加编号和顺序线 |
| 动画导致遮挡核心内容 | 允许重排 |
| 多个弹出层叠在一起 | 允许重排 |
| 最终态看不出讲解顺序 | 允许重排 |
| 页面信息太密 | 允许重排 |
| 一页包含多个逻辑组 | 允许重排或拆页 |

底线：不改原结论、不删核心图文、不凭空增加知识、保留 `base.pdf` 对照。

## 9. 拥挤判断

拥挤判断是算法关键。

| 指标 | 含义 |
|---|---|
| 对象覆盖率 | 文本/图形占页面面积比例 |
| 重叠率 | 对象之间互相遮挡比例 |
| 空白率 | 页面可放标注的空白区域大小 |
| 文本密度 | 字数、文本框数量、字号 |
| 动画目标数量 | 有多少对象参与动画 |
| 标注可放置性 | 编号、箭头、提示条能否放下且不遮核心内容 |

## 10. 新增页规则

| 情况 | 是否新增页 |
|---|---:|
| 能在原页清楚融入动画导读 | 不新增 |
| 标注会遮挡核心内容 | 新增 |
| 一个页面有多个逻辑组 | 新增 |
| 动画步骤超过 5 个且无法合并 | 新增 |
| 有明显遮挡，需要展示被盖住内容 | 新增 |
| 原页本身已经很拥挤 | 新增 |
| 只是 1-2 个小变化 | 不新增 |

单页最多生成 4 页。极复杂页进入报告，提示建议人工拆课件。

## 11. AI 接入边界

| 功能 | 动画导读基础版 | AI 讲义解释版 |
|---|---:|---:|
| 识别动画顺序 | 做 | 做 |
| 识别遮挡/拥挤 | 做 | 做 |
| 标注变化顺序 | 做 | 做 |
| 解释每页知识点 | 不做 | 做 |
| 推理动画讲解逻辑 | 不做 | 做 |
| 生成复习提示 | 不做 | 做 |
| 无依据扩写 | 禁止 | 禁止 |

基础版不依赖 AI API。AI 版必须可关闭、标来源，依据不足时写“需教师补充”。

## 12. 技术架构

| 模块 | 作用 | 技术 |
|---|---|---|
| 上传服务 | 接收 `.pptx`，创建任务 | Python Web 服务 |
| 原生转换器 | 生成 `base.pdf` | LibreOffice headless，后续可换 Unoserver |
| PPTX 解析器 | 读取对象、动画、备注、层级 | OOXML 直读 / python-pptx 辅助 |
| 页面分析器 | 判断动画复杂度、遮挡、拥挤、多逻辑组 | 自研规则算法 |
| 重排决策器 | 决定原生增强、融合重排、少量拆页 | 规则引擎 |
| PDF 增强器 | 生成 `guide.pdf` | PyMuPDF / reportlab / pypdf |
| AI 解释器 | 后续生成 `ai_notes.pdf` | 可插拔 AI API |
| 前端 | 上传、预览、下载 | 本地 Web 工具界面 |

核心数据流：

```text
PPTX -> base.pdf + analysis.json -> augment_plan.json -> guide.pdf + report.json
```

## 13. 不支持边界

| 类型 | 首版处理 | 后续方向 |
|---|---|---|
| `.ppt` 老格式 | 不支持 | 视需求接转换链路 |
| Keynote | 不支持 | 视需求接转换链路 |
| 加密 PPTX | 不支持 | 直接提示 |
| 宏/VBA | 忽略并提示 | 暂不规划 |
| 音频/视频 | 不嵌入 PDF，只保留静态封面并提示 | 转写、关键帧、讲解摘要 |
| 复杂路径动画 | 不还原，只进入问题报告 | 路径说明、轨迹可视化 |
| 点击触发/交互动画 | 不还原，只做可检测提示 | 交互顺序还原和文字说明 |
| 字体缺失 | 原生转换并写入报告 | 提供字体修复建议 |
| 转换失败 | 直接报错 | 不生成假结果 |

## 14. 开箱即用封装

最终必须面向小白封装，不能要求用户走代码流程。

| 阶段 | 用户体验 |
|---|---|
| 开发期 | 本地 Web 服务，方便调试 |
| Demo 交付 | 双击启动脚本，自动打开浏览器 |
| 比赛展示 | 一个文件夹或压缩包，内含启动器和样例 |
| 最终产品 | 桌面应用 / 本地 Web App / 云端 Web 服务 |

小白流程：双击启动 -> 浏览器自动打开 -> 拖入 `.pptx` -> 选择标准版或动画导读版 -> 下载结果。

## 15. 开发阶段和验收标准

| 阶段 | 开发内容 | 验收标准 |
|---|---|---|
| V3A | 原生转换底座 | `.pptx` 能转出 `base.pdf`，排版和图片基本保留 |
| V3B | 动画分析 | 能输出每页动画对象、顺序、遮挡、拥挤程度 |
| V3C | 动画导读页生成 | 能生成 `guide.pdf`，动画融入页面，不只是旁边说明 |
| V3D | 重排决策 | 能区分原生增强、融合重排、少量拆页 |
| V3E | 前端封装 | 小白能上传、选择模式、下载结果 |
| V3F | 样例验证 | 至少 3 份不同类型 PPTX 跑通：课程、培训、图文密集 |

首版完成标准：

| 必须满足 | 标准 |
|---|---|
| 保真 | `base.pdf` 基本等同普通 PPT 转 PDF |
| 动画导读 | `guide.pdf` 能体现动画顺序变化 |
| 不爆页数 | 不按动画数量无限拆页 |
| 可对照 | 用户能下载 `base.pdf` 和 `guide.pdf` |
| 可用 | 非技术用户能通过网页完成转换 |
| 诚实 | 不支持项进入 `report.json`，不静默失败 |
