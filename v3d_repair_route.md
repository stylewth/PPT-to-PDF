# V3D 修复路线：复杂课件可读性

## 结论

当前问题不是后续才处理的小问题，而是 V3D 必须先修的核心可读性问题。目标从“能标出来”改为“标得有用、少打扰、能解释为什么加页”。

## 修复原则

| 原则 | 定法 |
|---|---|
| 原页优先 | `base.pdf` 永远保留，`guide.pdf` 默认接近原 PPT 页数 |
| 少标注 | 不再大面积画框，默认只显示 1-3 个编号锚点 |
| 不懂就不乱加页 | 复杂页无法可靠解释时写入 `report.json`，不生成一堆低价值导读页 |
| 页数有预算 | 默认导读版页数不超过源 PPT 的 1.15 倍；超过则转为摘要/报告 |
| 可读性先于算法炫技 | 文本溢出、遮挡正文、标注堆叠都视为失败 |

## 当前证据

| 项 | 结果 |
|---|---|
| 测试文件 | `赛道3/app/samples/Review+chapter24-27.pptx` |
| 源 PPT 页数 | 42 |
| 当前 `guide测试.pdf` 页数 | 75 |
| 主要问题 | 绿色框意义不清、遮挡原内容、导读页堆叠、导读页文本溢出 |

## 阶段 1：止损热修（已完成）

| 任务 | 改动 | 验收 |
|---|---|---|
| 关闭大面积绿色框 | `pdf_augmenter.py` 默认不画完整矩形框，只保留小编号徽标 | 已完成：Review 样例原页不再出现覆盖大段文字的大框 |
| 限制标记数量 | 每页最多 3 个锚点，优先标动画组首项/关键对象 | 已完成：计划层和生成层都限制为最多 3 个锚点 |
| 禁止无意义导读页泛滥 | `expand_after_native` 不再默认每页追加导读页，改为只写入报告 | 已完成：Review 样例 `guide.pdf` 从 75 页降为 42 页 |
| 加回归测试 | 新增 Review 样例计划测试，不跑完整 PDF 也要验证页数预算 | 已完成：`guide_page_count` 不再等于复杂页数量 |

## 阶段 2：有用标注

| 任务 | 改动 | 验收 |
|---|---|---|
| 标注语义升级 | `inline_markers` 增加 `role`: `first_change` / `covered_content` / `key_result` | 用户能从编号知道“先看哪里” |
| 弱高亮 | 只对小对象/遮挡对象用淡色细边或半透明底，不画整页横跨框 | 不遮挡正文和公式 |
| 页内提示条 | 有空白才加底部/右侧一句话提示；无空白则不加 | 提示条不压住 PPT 主内容 |

### 当前进展

| 已完成 | 说明 |
|---|---|
| 标注语义升级 | `inline_markers` 已增加 `role` 和 `hint`，支持 `first_change`、`covered_content`、`key_result` |
| 遮挡识别依据 | 分析层按对象边界框和 z-order 判断动画对象是否覆盖已有对象，不再只看动画顺序 |
| 页内小提示条 | 原生页内会在编号旁加入“先出现 / 遮挡变化 / 关键结果”等短提示，仍保持最多 3 个 |

## 阶段 3：导读页重做

| 任务 | 改动 | 验收 |
|---|---|---|
| 只给必要页加导读页 | 条件改为：明显遮挡核心内容、动画步骤可合并成 1 个清晰组、页面有足够摘要价值 | Review 样例不会连续几十页导读 |
| 分组摘要 | 多个动画对象合并成“本页 3 个变化组”，不逐动画列清单 | 导读页不是流水账 |
| 防溢出版式 | 每页最多 4 条，每条最多两行，超出写“更多见报告” | 截图无文字越界 |

## 阶段 4：拥挤判断重构

| 任务 | 改动 | 验收 |
|---|---|---|
| 新增 `layout_decider.py` | 把策略从 `augment_planner.py` 中拆出：判断 `keep_native` / `native_enhance` / `report_only` / `expand_after_native` | 每页策略有原因 |
| 标注可放置性 | 计算上/下/左/右空白区，决定能不能放提示条 | 没空间就不硬塞 |
| 全局页数预算 | 按整份 PPT 控制新增页数量，而不是单页各自决定 | `guide.pdf` 页数可控 |

## 阶段 5：Review 样例验收

| 验收项 | 标准 |
|---|---|
| 页数 | 默认模式下 `guide.pdf` 不超过源 PPT 1.15 倍 |
| 标注 | 单页最多 3 个编号锚点，不能遮挡主要文字/公式 |
| 导读页 | 不能出现文本溢出；不能连续大量出现低价值清单页 |
| 报告 | 无法可靠处理的复杂页进入 `report.json`，明确说明原因 |
| 原生保真 | `base.pdf` 仍由 LibreOffice 原生转换，作为对照文件保留 |

## 不在本轮做

| 不做 | 原因 |
|---|---|
| AI 推理动画意图 | 基础版先解决可读性，不靠 AI 遮丑 |
| 完美重排所有复杂页 | 范围过大，先做可解释策略和有限样例 |
| 复杂路径动画还原 | 当前阶段先报告，不做伪还原 |
