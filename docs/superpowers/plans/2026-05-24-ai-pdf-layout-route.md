# 2026-05-24 AI 解释融入 PDF 排版路线

## 目标

| 项 | 定法 |
|---|---|
| 输出关系 | `base.pdf` 保留原生对照，`guide.pdf` 保留基础学习版，AI 解释只生成独立 `ai_guide.pdf`。 |
| 输入来源 | 只使用 Web 中已生成并通过来源审计的块级 AI 解释。 |
| 安全边界 | 导出 PDF 不接收、不保存、不转发 API key；只接收 `job_id` 和解释内容。 |
| 比赛价值 | 展示“用户按需选择知识块 -> Agent 逐块解释 -> 可沉淀成学习资料”的闭环。 |

## 方案对比

| 方案 | 做法 | 优点 | 风险 | 当前取舍 |
|---|---|---|---|---|
| A. 原页后插 AI 解释页 | 每个原 guide 页后追加对应 AI 解释页 | 稳、快、不覆盖原文，长解释也能放下 | 页数增加，原文和解释不在同一页 | 当前 MVP 已采用 |
| B. 原页编号 + 后页详解 | 原 guide 页叠小编号，后一页按编号解释 | 对应关系更清楚，仍不挤压原文 | 需要安全编号落点和截图门禁 | 下一步推荐 |
| C. 页边短旁注 | 在原页空白区放 1-2 行解释 | 阅读连贯，像真正讲义 | 容易盖住正文、公式或图 | 只适合短解释 |
| D. 页内融合重排 | 局部移动原对象，为解释腾空间 | 最像最终成品 | 算法和视觉门禁要求高，容易路线偏移 | 最后做 |

## 分阶段路线

| 阶段 | 目标 | 验收 |
|---|---|---|
| V5H 简单 AI PDF | 复制 `guide.pdf`，在有解释的源页后插入 AI 解释页 | 能生成 `ai_guide.pdf`；文本可抽取；manifest 记录源页和解释页 |
| V5I 编号锚点版 | 在原页选中块附近加小编号，后一页按编号解释 | 编号不盖文字/图/公式；截图通过；manifest 有编号与块映射 |
| V5J 短旁注版 | 对很短解释尝试放入原页空白区 | 只允许短文本；无安全空白则回到插页方案 |
| V5K 融合重排版 | 对部分页面做局部让位，把解释和原文整合在同页 | 不改变 `base.pdf`/`guide.pdf`；视觉门禁和人工截图均通过 |

## 排版算法原则

| 原则 | 规则 |
|---|---|
| 先保原文 | AI 文本不能覆盖原 PPT 原文、公式、图和媒体关键帧。 |
| 先短后长 | 短解释可尝试页内旁注，长解释默认插页。 |
| 先局部后全局 | 只围绕选中知识块处理，不因 AI 文本重排整页。 |
| 无空间就插页 | 任何安全空白不足、文字溢出、视觉门禁失败，都回到插页。 |
| 来源可追溯 | 每段解释必须保留 `source_refs`，manifest 记录 block、源页、输出页。 |

## Manifest 设计

| 字段 | 含义 |
|---|---|
| `source_page` | 原 `guide.pdf` 页码 |
| `explanation_page` | AI 解释所在页码 |
| `block_ids` | 本页写入的知识块 |
| `version` | 与知识块 prompt/schema 版本一致 |
| 后续扩展 | `anchors`、`note_boxes`、`layout_mode`、`visual_check` |

## 质量门禁

| 检查 | 标准 |
|---|---|
| 来源审计 | 导出前再次校验解释来源必须属于对应块。 |
| PDF 文本 | 生成页可抽取出 AI 解释和关键内容。 |
| 页数关系 | AI 版页数 >= guide 页数，且不覆盖 guide 原页。 |
| 截图门禁 | 后续编号/旁注/融合阶段必须渲染 PNG 检查。 |
| 安全检查 | 输出文件和缓存不能包含 API key、Base URL 或请求头。 |
